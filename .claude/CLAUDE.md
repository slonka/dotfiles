- Use "-" instead of "—".
- Nothing should ever be changed (POST, PATCH, DELETE, not only HTTP but others as well) outside of this machine when in plan mode.
- If need access to github use `gh`.
- PR reviews: always use `gh api` to create PENDING reviews with line-level code comments. Never use `gh pr review --approve` (submits immediately). Never approve without my explicit confirmation. When creating reviews via `gh api`, omit the `event` field - reviews with comments are pending by default.
- Development in repos with multiple contributors or when multiple claude processes are running in the same project must happen in a git worktree. Create new worktrees under `.trees/<branch>` from the git root. Come up with really short branch name based on my input and/or issue linked.
- Do not amend to commits, use "-sS" flags, create new ones when asked to, never push yourself
- To type formatted text into Slack, use `~/bin/slack-type "mrkdwn text"` or `slack-type -f file.md`. Slack only renders mrkdwn formatting when typed, not pasted. Default 3s delay before typing (`-d` to change). User must focus the Slack input field during the delay.
- Keep Bash commands simple and avoid piping (e.g. `| tail`, `| head`, `| grep`). Piped commands form a single permission string, so approving `gradle:*` won't match `./gradlew build | tail -10`. Run the base command on its own and use dedicated tools (Read with offset/limit, Grep) to inspect output if needed.
- Always use full URLs with `https://` prefix so they are clickable in the terminal.
